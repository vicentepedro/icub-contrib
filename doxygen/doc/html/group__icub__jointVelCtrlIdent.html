Identification of the joint plant when controlled in velocity mode.

Description

This module allows for system identification of the joint when controlled in velocity mode. Ideally, the plant should be represented by a pure integrator, but the behavior might deviate from the baseline due to the presence of poles and zero. To this end the joint is controlled with a chirp profile in velocity whose output is then sent to a yarp port for logging purpose.

The joint is controlled in three different modalities that follow one another:

  • mode 0: the joint is subject to the chirp signal in velocity.
  • mode 1: the joint is steered to reach for a target value under the action of a minimum-jerk controller that does not account for plant dynamics.
  • mode 2: the joint is steered to the target under the action of a minimum-jerk compensator that accounts for the identified plant dynamics.

Parameters

–name name

  • specify the module stem-name.

–robot robot

  • select the robot to connect to.

–period Ts

  • specify the controller period given in [ms].

–part part

  • select the part to control. The special part "head_v2" is also available.

–joint joint

  • select the joint to control.

therefore the following parameters can be assigned through command-line options: gain, fmin, fmax, T.

The model of the identified plant is:

therefore the following parameters can be assigned through command-line options: Kp, Tz, Tw, Zeta.

Moreover, the reactivity of the minimum-jerk controllers can be tuned by changing their point-to-point trajectory time through the Ttraj command-line parameter.

User might want also to tune the low-level PID controller, and to this purpose the following command-line options are available: PID_Kp, PID_Ki, PID_Kd.

Ports Created

/ <modeName>:o streams out the following data:

<cnt> <vel> <ref> <pwm_1> [... <pwm_n>] <fb> <dfb>

where cnt is a progress number accounting for the mode type such that mode=mod(cnt,3), vel is the commanded velocity in [deg/s], ref is the reference position as integrated by the board given in [deg], pwm is the pid output in [V] and fb represents the joint encoder feedback in [deg]. The last term dfb is the estimated derivative of the feedback.
Importantly, the quantity pwm is retrieved only if the low level board broadcasts it out.
